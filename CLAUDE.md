# General

- Git Hooks: pre-commit checks formatting and linting, pre-push checks the build and the tests.
- Prefer modularity, several short files is better than few big files.
- Create code easy to read and maintain for a junior dev that discovers both the project and the stack.

# Frontend

- La logique n'est jamais dans les composants (React).
- The logic is never written inside components (React).
- Prefer tailwindcss, shadcn, nuqs and next.js.
- Write domain logic in `/features/[domaine name]/`.
- Prefer modularity with small files.
